# Release v1.1.0 — Transacciones por Voz

**Fecha**: 2026-03-25
**Branch**: `claude/setup-cicd-pipeline-0GpMb`
**Commits**: 5

## Resumen

Reemplazo completo del sistema de IA predictiva por transacciones basadas en voz.
El usuario ahora puede dictar ventas y compras en lenguaje natural colombiano,
y la IA las convierte en transacciones estructuradas para confirmar.

## Nuevas funcionalidades

### Registro por voz (Mobile)
- Botón de micrófono en dashboard y en pantalla dedicada `/voice-transaction`
- Reconocimiento de voz on-device (privacidad: audio nunca sale del teléfono)
- Fallback automático de locale: `es_CO` → `es_*` → default
- Feedback visual del nivel de sonido (mic pulsa con la voz)
- Timeouts configurados: 30s escucha máxima, 5s pausa para auto-stop
- Pantalla de confirmación con resumen de la transacción parseada
- Chip de confianza: "Alta confianza" (verde) / "Verificar datos" (naranja)

### Parsing con Claude API (Backend)
- Modelo: Claude Haiku 4.5 (~$0.001/transacción, <200ms)
- Método: `tool_use` con `tool_choice: forced` → JSON garantizado
- Prompt caching: `cache_control: ephemeral` → 90% ahorro
- Temperature: 0 (determinístico)
- Catálogo de productos del equipo inyectado en system prompt
- Matching fuzzy de productos por nombre
- Soporte jerga: "lucas", "barras", "quina", "fiado", "nequi", "daviplata"

### Seguridad
- Prevención de prompt injection (11 regex patterns + delimitadores)
- Input sanitización: 3-500 chars, strip de markup/code blocks
- Rate limit: 5 req/min por usuario en endpoint AI
- max_tokens: 512 (limita costo y previene output excesivo)
- Validación de cantidades: 1-10,000 unidades
- API key solo en backend, nunca expuesta al cliente

## Breaking changes

- Eliminado: `AiInsight`, `AiPriceSuggestion`, `AiDemandForecast`
- Ruta cambiada: `/ai-chat` → `/voice-transaction`
- Requiere `ANTHROPIC_API_KEY` en variables de entorno del backend
